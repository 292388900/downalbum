首先介绍一下编码和文件格式

日常见到的文件，譬如.avi, .rmvb, .mpg, .mp4等。这些都是文件格式，是各个厂商或者平台为了能在自己的系统上播放，或者保护视频等原因制定的。文件格式繁多！据不完全统计统计至少在30种以上。这种东西移植性比较差。


但是文件中存放的视频，其编码又是多样的，但主流的比较有限，

1. rm --（real公司的保护格式)

2. h264 也叫做 AVC编码  -- 目前最高级的视频压缩编码方案，分成好几个等级，默认普通等级就行）

3. mpeg4  -- 上一代压缩标准

4. xvid及divx -- 这两个都是变种的mpeg4。

5. mpeg2 -- svcd/dvd标准，也就是我们看到的质量比较好的，但是体积比较大的

6. mpeg1 -- vcd标准，主要是解码速度快，对CPU占用小。


视频压缩比比较

mpeg2 <= mpeg1 < mpeg4 <= xvid <= divx <= rmvb <= h264


但是各家编码都是各有所长，也就是不同类型的视频（包括视频中的物体的运动程序程度，色调昏暗等等）
总体来说，h264 官方认可的压缩比最大。


也就是说，avi文件中也可以保存h264。
我们180上的电影，为了要求质量，通常是avi + mpeg2，所以一部电影通常是700M。
但是如果 avi文件中本身就是h264编码，压缩出来不一定很大。
目前avi中多用xvid+2pass编码！也有用mpeg4编码的。

-----------------------------------------
打个比喻，文件格式 类似于数据库系统
kbase, mysql, sql server, oracle

视频编码(video codec)类似 数据库中数据，你可以把数据存在任何的数据库中。

但是很多系统仅支持有限的数据库，譬如 psp, ipod等仅仅支持mp4文件格式，或者他们自定义的格式！

市场份额这块：



比较常用的功能首先是移动视频，也是市场份额最大的一块，一个是手机市场日趋完善，另一个是移动媒体异军突起。
同时目前旧有的视频，以前的视频，不兼容，体积大，转换的需求比较多。所以这块是主流。

--------------------------------------------
常见的手机，但是手机视频分成3gp, 3gpp, mp4，

3gp压缩率较小，体积占用比较大，但是视频解码速度快，主要是受当时手机 CPU限制。同时分辨率限制在176*144，分辨率非常小。

3gpp比3gp进行了改进，但改进不大，主要是分辨率可以任意调整。

mp4压缩比比较大，但是一般的手机支持的不知很完善。最新的高级手机一般支持。


手机还有一种需求，就是只保留视频声音，主要是压缩相声，百家讲坛这种，以声音为主导的。
也需要考虑纯音频转换，譬如将 mp3 转换成 AAC 编码的音频，AAC比mp3压缩比更大，质量更好。
